What it does

⇄

Any layouts

Not hard-coded to a pair — it converts over your actually-installed layouts, via physical key positions. So ß/æ → ы/э just works.

⌨️

Your hotkey

Select the garbled text (or none — it grabs back to line start) and trigger your hotkey; the input source flips so you keep going. Set it in Settings — a combo, a modifier tap/chord (e.g. double-tap Shift), or a tap sequence.

✨

Auto-correct (optional)

A trigram language model fixes wrong-layout words as you type, no hotkey. Cross-script, precision-first, default off.

↩︎

Press-again undo

Hit the hotkey again within ~1.5 s to restore the original text and the previous input source.

🤫

Reads quietly

Accessibility-first read/write — no clipboard churn, so pop-up translators like DeepL stay out of your way.

🚫

Per-app exceptions

Keep auto-correct off in terminals, IDEs, or any app you choose. Password fields are always skipped.

🔄

Auto-updates

Signed, notarized, and kept current via Sparkle. Or install & update through Homebrew.

Install

Homebrew

brew install --cask vladforfutdinov/relayout/relayout

Updates with brew upgrade.

Direct download

Grab reLayout.dmg from the latest release and drag it to Applications.

  1. Launch reLayout — it lives in the menu bar (the rL icon).
  2. Grant System Settings → Privacy & Security → Accessibility, then relaunch.
  3. Select wrong-layout text and tap the hotkey (default: left ⌥).
